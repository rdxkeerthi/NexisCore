# Security Policy

## Supported Versions

NexisCore is actively maintained. Security updates are provided for the latest stable release.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

Security is a core focus of NexisCore. If you discover a security vulnerability, we would appreciate it if you could report it to us privately so that we can fix it before it is exploited.

Please **do not open a public issue** for security vulnerabilities.

Instead, please send an email to:
**`keerthivasanm.student@saveetha.ac.in`**

Include the following in your report:
* Type of issue (e.g., buffer overflow, SQL injection, cross-site scripting, etc.)
* Full paths of source file(s) related to the manifestation of the issue
* The location of the affected source code (tag/branch/commit or direct URL)
* Any special configuration required to reproduce the issue
* Step-by-step instructions to reproduce the issue
* Proof-of-concept or exploit code (if possible)
* Impact of the issue, including how an attacker might exploit the issue

We will endeavor to respond to your report within 48 hours. If the vulnerability is confirmed, we will work with you to patch the issue and release a security update.

Thank you for helping keep NexisCore secure!
